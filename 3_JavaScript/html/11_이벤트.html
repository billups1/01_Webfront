이벤트

        이벤트: 동작, 행위
        ->브라우저에서 할 수 있는 동작

        ex) click, keyup, keydown, keypress, mouseover, mounseon,...

        아래에서 확인
        https://www.w3schools.com/js/js_events.asp

        이벤트 리스너
        -> 브라우저에서 이벤트가 감지 될 때까지 대기하다가
        감지가 되었을 때 연결된 기능(함수)를 수행하는 것
        -> "on이벤트명" 형식으로 작성됨
        
        ex) onclick, onchange, onsubmit, onkeyup, ...


        이벤트 핸들러
        -> 이벤트 리스너에 연결된 기능(함수)로
        이벤트가 감지되었을 때 수행할 내용을 작성한 함수

인라인 이벤트 모델

        * html 요소에 이벤트리스너="이벤트핸들러"
        형식을 작성하는 방법

        * 하나의 요소에 동일한 이벤트 핸들러를 중복 작성 x
        (이전에 작성된 이벤트 핸들러를 새로운 이벤트 핸들러가 덮어쓰기함)

        * 최근에는 많이 사용을 하진 않지만 종종 사용함

        * 인라인 이벤트 모델 작성 시 this
        -> 이벤트가 발생한 현재 요소

고전 이벤트 모델

        * html 요소에 이벤트 관련 코드를 작성하지 않고
        js에서 dom 객체에 직접 이벤트 관련 코드를 작성하는 방법

        * 하나의 요소에 동일한 이벤트를 핸들러를 중복 작성 x

표준 이벤트 모델( 요소.addEventListner() )

        - w3c(웹표준 재정단체)에서 공식적으로 지정한 이벤트 모델

        - 장점 : 한 요소에 중복되는 이벤트 리스너를 만들어
        여러 이벤트 핸들러를 추가할 수 있다. 
        (고전 이벤트 리스너 모델 단점 해결)
